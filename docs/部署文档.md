# StuLink — 部署文档

## 部署方式总览

| 方式 | 适用场景 | 难度 |
|------|---------|------|
| Docker 一键部署 | NAS / Linux 服务器 / 本地快速启动 | ⭐ 简单 |
| 阿里云 ECS 部署 | 生产环境 / 公网访问 | ⭐⭐ 中等 |

---

## 一、Docker 部署（推荐）

### 1.1 部署流程

部署仅需两个文件（位于 `deploy/` 目录）：

```
deploy/
├── stulink-v1.4.5.tar    # Docker 镜像（62MB，含全部应用代码）
└── docker-compose.yml    # 容器编排配置
```

**步骤：**

```bash
# 1. 将 deploy/ 目录上传到服务器的任意位置，如 /opt/stulink/

# 2. 导入镜像
cd /opt/stulink
docker load -i stulink-v1.4.5.tar

# 3. 启动容器（首次启动会自动创建数据库）
docker compose up -d

# 4. 确认运行
docker ps | grep stulink
```

首次启动后，系统会在 `stulink-data/` 目录下自动生成：
- `system.db` — 基础数据库
- `dormitory.db` — 宿舍数据库
- `history.db` — 历史记录库
- `.secret_key` — Flask 会话密钥
- `.encryption_key` — 身份证加密密钥

访问 `http://服务器IP:5000`，默认管理员 `admin` / `admin123`。

### 1.2 NAS 部署（绿联 UGOS）

**前置**：NAS 已开启 SSH（绿联 UGOS：设置 → 终端机 → 启用 SSH），已安装 Docker 套件。

#### ① Windows 上构建并导出镜像

```powershell
docker build -t stulink:v1.4.5 .
docker save stulink:v1.4.5 -o deploy/stulink-v1.4.5.tar
```

`deploy/` 目录产物：
```
deploy/
├── stulink-v1.4.5.tar    # 镜像（含全部应用代码）
└── docker-compose.yml    # 仅挂载数据目录
```

#### ② NAS 上创建目录

```bash
ssh root@你的NAS的IP -p 922
mkdir -p /volume1/docker/stulink
cd /volume1/docker/stulink
mkdir stulink-data
```

#### ③ 上传文件（SMB）

Windows 资源管理器地址栏输入 `\\你的NAS的IP`，将 `deploy/` 下两个文件上传到 `/volume1/docker/stulink/`。

#### ④ 导入镜像并启动

```bash
cd /volume1/docker/stulink
docker load -i stulink-v1.4.5.tar
docker compose up -d
docker ps   # 确认 stulink 状态为 Up
```

#### ⑤ NAS 上升级

```bash
cd /volume1/docker/stulink
docker compose down
docker rmi stulink:v1.4.5
# 通过 SMB 上传新版本 tar
docker load -i stulink-v2.0.0.tar
# 更新 docker-compose.yml 中的 image 版本号
docker compose up -d
```

### 1.3 升级方法（通用）

```bash
# 1. 停止旧容器
docker compose down

# 2. 替换镜像（删除旧镜像，导入新 tar）
docker rmi stulink:v1.4.5
docker load -i stulink-v2.0.0.tar

# 3. 更新 docker-compose.yml 中的 image 版本号

# 4. 启动
docker compose up -d
```

> ⚠️ `stulink-data/` 目录**绝对不能删除或覆盖**，所有业务数据都在其中。

### 1.4 常用运维命令

```bash
docker compose down          # 停止服务
docker compose up -d         # 启动服务
docker compose restart       # 重启服务
docker logs -f stulink       # 实时查看日志
docker exec -it stulink sh   # 进入容器 shell
```

### 1.5 自定义端口

修改 `docker-compose.yml` 中的 `ports` 即可，如改为 8080：

```yaml
ports:
  - "8080:5000"
```

## 二、阿里云 ECS 部署

### 2.1 前置条件

- 阿里云 ECS 实例（CentOS 7+ / Ubuntu 20.04+ / Alibaba Cloud Linux）
- 安全组已放行 **5000** 端口（或自定义端口）
- 已安装 Docker 和 Docker Compose

### 2.2 安装 Docker（CentOS 7 示例）

```bash
# 安装 Docker
yum install -y yum-utils
yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
systemctl enable docker
systemctl start docker

# 验证
docker --version
```

### 2.3 安装 Docker（Ubuntu 示例）

```bash
apt update
apt install -y docker.io docker-compose-v2
systemctl enable docker
systemctl start docker
```

### 2.4 上传部署文件

```bash
# 方法一：scp 上传（在本机执行）
scp -r deploy/ root@你的ECS公网IP:/opt/stulink/

# 方法二：使用阿里云 OSS 中转
# 先将 deploy/ 上传到 OSS，再在 ECS 上下载
```

### 2.5 部署启动

```bash
cd /opt/stulink
docker load -i stulink-v1.4.5.tar
docker compose up -d
```

### 2.6 配置安全组

在阿里云控制台 → 网络与安全 → 安全组 → 添加规则：

| 方向 | 协议 | 端口 | 授权对象 |
|------|------|------|---------|
| 入方向 | TCP | 5000 | 0.0.0.0/0 |

### 2.7 配置域名（可选）

1. 在 DNS 服务商将域名 A 记录指向 ECS 公网 IP
2. （推荐）使用 Nginx 反向代理，实现 80 端口访问 + HTTPS：

```nginx
# /etc/nginx/conf.d/stulink.conf
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

```bash
# 安装并启动 Nginx
yum install -y nginx    # CentOS
apt install -y nginx    # Ubuntu
systemctl enable nginx
systemctl start nginx

# 测试配置
nginx -t
systemctl reload nginx
```

### 2.8 配置 HTTPS（推荐）

使用 Certbot 免费 SSL 证书：

```bash
# CentOS 7
yum install -y certbot python3-certbot-nginx
# Ubuntu
apt install -y certbot python3-certbot-nginx

# 自动配置 Nginx + SSL
certbot --nginx -d your-domain.com

# 设置自动续期
echo "0 3 * * * root certbot renew --quiet" >> /etc/crontab
```

### 2.9 配置开机自启

`docker-compose.yml` 已设置 `restart: unless-stopped`，Docker 服务自启即可：

```bash
systemctl enable docker
```

## 三、生产环境配置建议

### 3.1 修改默认密码

首次登录后立即修改 admin 密码：右上角 → 改密码。

### 3.2 设置 SECRET_KEY

编辑 `docker-compose.yml`，将 `SECRET_KEY` 设为强随机值：

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
# 将输出填入 docker-compose.yml 的 SECRET_KEY=
```

### 3.3 数据备份

```bash
# 手动备份（复制整个数据目录）
cp -r stulink-data stulink-data-backup-$(date +%Y%m%d)

# 定时备份（添加到 crontab，每天凌晨 2 点）
0 2 * * * cp -r /opt/stulink/stulink-data /opt/stulink/backups/stulink-data-$(date +\%Y\%m\%d)
```

### 3.4 资源要求

| 指标 | 最低 | 推荐 |
|------|------|------|
| CPU | 1 核 | 2 核 |
| 内存 | 512MB | 1GB |
| 磁盘 | 1GB（不含数据） | 10GB+ |
| 系统 | CentOS 7+ / Ubuntu 20.04+ | Alibaba Cloud Linux 3 |

## 四、故障排查

**Q: 容器启动后立即退出？**
```bash
docker logs stulink  # 查看错误日志
```

**Q: 5000 端口被占用？**
修改 `docker-compose.yml` 中 `ports` 为其他端口，如 `"8080:5000"`，同时更新安全组规则。

**Q: 数据库被锁定（database is locked）？**
SQLite 仅支持单进程写入，确保只有一个 stulink 容器在运行：
```bash
docker ps -a | grep stulink
docker rm -f stulink  # 清理旧容器后重新启动
```

**Q: 外网无法访问？**
1. 检查安全组是否放行端口
2. 检查防火墙：`firewall-cmd --add-port=5000/tcp --permanent && firewall-cmd --reload`
3. 确认容器正在监听 `0.0.0.0:5000`：`docker logs stulink`

**Q: 忘记 admin 密码？**
在宿主机上运行：
```bash
docker exec -it stulink python scripts/init_db.py
```
> ⚠️ 此操作会重置 admin 密码为 admin123，但**不会**影响已有数据。
