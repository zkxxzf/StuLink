# StuLink — 智联校园学生管理系统 · 设计文档

## 1. 项目概述

**StuLink（智联校园学生管理系统）** 是一套面向中学的综合学生管理平台，采用双站点架构：在校生管理系统（主应用 :5000）和往届生查询系统（独立应用 :5001）。基于 Python Flask 构建，多文件 SQLite 分库架构，支持 Docker Compose / 阿里云 ECS / NAS / Windows 本地部署。

## 3. 核心模块设计

### 3.1 认证与权限

**用户模型** (`User`):
- 角色: `admin` / `dorm_manager` / `homeroom_teacher` / `grade_leader` / `school_viewer`
- 密码: Werkzeug `generate_password_hash` (scrypt)

**权限控制**:
- `@login_required` — Flask-Login 会话保护
- `@role_required('admin', ...)` — 路由级拦截
- 登录频率限制: 每 IP 每 60 秒最多 10 次
- 重定向验证: 防止开放重定向
- SECRET_KEY: 自动生成并持久化到 `data/.secret_key`

**安全措施**:
- CSRF: Flask-WTF 全局启用
- Session: `HttpOnly=True`, `SameSite=Lax`
- `nl2br` 安全过滤器（先转义再换行，替代危险的 `|safe`）
- body `padding-top: 56px` 避免固定导航栏遮挡

### 3.2 侧边栏按模块显示

| 所在模块 | 侧边栏内容 |
|----------|-----------|
| 欢迎页 | 无侧边栏（全宽布局） |
| 宿舍管理 | 学生查询 → 宿舍管理（列表/分配）→ 床位管理（床位分配）→ 宿舍总览 → 统计报表 |
| 系统管理 | 系统管理 → 学生管理 / 教师管理 / 字典管理 / 班型设置 / 权限组管理 / 年级管理 |
| 积分/成绩 | 返回首页 + 学生查询 |

侧边栏顶部链接按模块显示对应名称（宿舍管理/系统管理/积分管理/成绩管理），点击回到该模块概览页。

往届生查询系统位于独立应用（:5001），有自己的导航栏（基本信息查询 / 宿舍查询标签页），不与主应用侧边栏混合。

### 3.3 身份证号加密

**AES-256-CBC 确定性加密**（同一明文→同一密文，支持等值查询）
- 密钥: 环境变量 `ENCRYPTION_KEY` → `data/.encryption_key`
- Model 层透明加解密 (`@property` getter/setter)
- `decrypt()` 自动识别旧明文数据

### 3.4 宿舍自动分配算法 (V4)

两阶段 + 7 种情形逐级尝试，支持 6/8 人间，合班分配，楼层连续性优化。

## 5. 数据库设计

### 5.1 多文件分库架构

| 数据库文件 | SQLAlchemy Bind | 内容 |
|-----------|----------------|------|
| `system.db` | default | 用户、学生基础信息、字典、权限组、班型、年级（基础库） |
| `dormitory.db` | dormitory | 房间、床位分配、学生住宿信息（宿舍库） |
| `history.db` | history | 分配变更流水、操作日志（历史库） |

### 5.2 模块数据分离设计

根据学校管理业务流程，各模块数据按以下原则分离存储：

| 模块 | 前端入口 | 基础数据来源 | 模块专属数据存储 |
|------|---------|------------|----------------|
| 系统管理 | `/system/` | `system.db` | `system.db` |
| 宿舍管理 | `/dashboard/` | `system.db`（学生基础信息） | `dormitory.db`（住宿信息、房间、床位） |
| 成绩管理（待开发） | `/grades/` | `system.db`（学生基础信息） | `grades.db`（成绩数据） |
| 积分管理（待开发） | `/points/` | `system.db`（学生基础信息） | `points.db`（积分与奖惩数据） |

**设计原则**：
- 学生基础信息（姓名、学号、性别、身份证号、民族、年级、班级、毕业学校等）统一存储在 `system.db` 的 `students` 表中，作为各模块的共享数据源
- 各模块专属数据（住宿信息、成绩、积分等）存储在各自的数据库中
- 跨库关联通过 `students.id` 建立，使用 SQLAlchemy `primaryjoin` + `viewonly` 维护应用层关系

### 5.3 核心数据表

**system.db**:
- `users` — 用户表（角色: admin / dorm_manager / homeroom_teacher / grade_leader / school_viewer）
- `students` — 学生基础信息表（不含住宿相关字段）
- `dict_categories` / `dict_items` — 字典数据
- `permission_groups` — 权限组配置
- `class_profiles` — 班型配置
- `grade_settings` — 年级设置
- `user_class_links` — 用户与班级关联

**dormitory.db**:
- `rooms` — 宿舍房间信息
- `bed_assignments` — 床位分配记录
- `student_accommodation` — 学生住宿信息（boarding_type, day_student_type, textbook, teacher_notes）
- `assignment_history` — 分配变更历史

**history.db**:
- `operation_logs` — 操作日志
- `student_change_log` — 学生变更记录

**跨库关联**：所有模块通过 `students.id` 或 `student_number` 引用学生，跨库外键约束在应用层通过 SQLAlchemy `primaryjoin` + `viewonly` 维护。学号具有 UNIQUE 约束且不可为空。

## 6. 数据存储与备份

```
data/
├── system.db          ← 基础库（用户/学生/字典/权限）
├── dormitory.db       ← 宿舍库（房间/床位分配）
├── history.db         ← 历史库（分配变更流水，独立文件性能隔离）
├── .secret_key        ← Flask 会话密钥
├── .encryption_key    ← 身份证加密密钥
└── backups/           ← 毕业操作时自动备份
    ├── graduate_2025级_20260629_120000.db
    └── history_2025级_20260629_120000.db
```

**毕业操作**：清空该年级的宿舍和床位分配，同时将主库和历史库一并复制到 `backups/` 目录。

## 7. 部署架构

### 7.1 Docker 全镜像打包

应用代码直接打入 Docker 镜像，部署时仅需一个 tar 文件 + docker-compose.yml：

```
deploy/
├── stulink-v1.6.1.tar    # 镜像（含 Python 运行时 + 全部应用代码）
└── docker-compose.yml    # 仅挂载数据目录 ./stulink-data:/app/data
```

**与旧版卷挂载方式的差异**：
- 旧：镜像只含依赖，代码通过 `-v ./code:/app` 挂载
- 新：代码打包进镜像，升级时替换 tar + 重启容器即可，无需管理代码目录

### 7.2 运行服务器

生产环境使用 Waitress WSGI 服务器，单 worker（SQLite 文件锁限制）。
