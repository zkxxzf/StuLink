# 模块化重构与欢迎页 — 实现计划 (V3)

## 核心思路

**"文件搬家，接口不变"** — 蓝图名和 URL 前缀基本不动，仅重组文件物理位置。新增 `welcome` 蓝图承载模块卡片导航页。

## 用户反馈变更（V1→V3）

| 变更项 | V1 | V3（最终） |
|--------|----|----|
| 学生管理归属 | dormitory | **system（第一栏）** |
| 批量删除学生 | 无限制 | **仅 admin** |
| 系统管理可见性 | 仅 admin | **所有人可见，无权时路由层 403** |
| 积分/成绩占位 | 有 | **有** |
| "更多模块上线"提示 | 有 | **移除** |

## 可行性结论

✅ **完全可行。** 蓝图名保持不变 → `url_for()` 引用零影响。URL 仅变更 1 处（`/` → welcome，`/dashboard` 承载仪表盘）。

## 关键设计决策

| 决策 | 说明 |
|------|------|
| 蓝图名不变 | `students`/`rooms`/`users` 等全部保持原名，url_for() 零影响 |
| URL 仅1处变更 | `/` 从 dashboard→welcome；`/dashboard` 承载原仪表盘 |
| dashboard 加 prefix | `url_prefix='/dashboard'` 避免与 welcome `/` 冲突 |
| 学生放 system | 学生基础信息跨模块共享，system 第一栏 |
| 系统管理全民可见 | 模板不隐藏，无权用户由 @role_required 返回 403 |
| 积分/成绩有占位 | 各自拥有 Blueprint + 路由 + 模板，展示"敬请期待" |

## 分阶段步骤

### 阶段1 欢迎页 + 占位模块
- 创建 `modules/welcome/routes.py`（Blueprint `welcome`, `GET /`）
- 创建 `templates/welcome/index.html`
- 创建 `modules/points/routes.py`（Blueprint `points`, `/points`, 渲染 `points/index.html`）
- 创建 `modules/grades/routes.py`（Blueprint `grades`, `/grades`, 渲染 `grades/index.html`）
- 创建 `templates/points/index.html`、`templates/grades/index.html`（"该模块正在开发中"）
- 修改 `auth.py`：重定向 → `url_for('welcome.index')`
- 修改 `base.html`：导航链接 → `url_for('welcome.index')`
- **不删旧文件，双套蓝图并行注册**

### 阶段2 dashboard 迁移
- 复制 `routes/dashboard.py` → `modules/dormitory/routes/dashboard.py`
- `Blueprint('dashboard', __name__, url_prefix='/dashboard')`
- welcome 页卡片链接 → `url_for('dashboard.index')`

### 阶段3 宿舍路由迁移
- 复制 `routes/rooms.py`、`assignments.py`、`statistics.py` → `modules/dormitory/routes/`
- 复制 `services/room_assignment_v4.py` → `modules/dormitory/services/`
- 修改 rooms.py 中 import 路径

### 阶段4 系统路由迁移
- 复制 `routes/students.py` → `modules/system/routes/students.py`（第一栏）
  - `batch-delete` → `@role_required('admin')`（已是，确认即可）
- 复制 `routes/users.py`、`dictionary.py`、`class_profile.py`、`perm_groups.py`、`grade_mgmt.py`

### 阶段5 模板重组
- 移动模板到模块目录，更新 `render_template()` 路径
- `base.html` 侧边栏：学生管理移入系统管理组；移除系统管理的角色隐藏包裹

### 阶段6 清理
- 删除 `app/routes/`；删除旧模板空目录
- `app/__init__.py` → `from app.modules import register_blueprints`
- 完整回归测试

## 欢迎页设计

3 模块卡片 + 系统管理入口：

```
┌──────────────────────────────────────────────────────┐
│           欢迎，{current_user.real_name}              │
│         请选择要使用的功能模块                         │
├──────────────────┬──────────────────┬────────────────┤
│  🏠 宿舍管理       │  ⭐ 积分管理       │  📊 成绩管理    │
│  宿舍分配/床位管理  │  该模块正在开发中   │  该模块正在开发中│
│  统计报表          │                   │               │
│  [进入模块]        │  [进入模块]        │  [进入模块]     │
├──────────────────┴──────────────────┴────────────────┤
│  ⚙ 系统管理 (所有人可见)                              │
│  学生管理 · 用户管理 · 字典配置 · 权限组 · 年级管理      │
│  [进入系统管理] → url_for('students.list_students')    │
└──────────────────────────────────────────────────────┘
```

- 积分/成绩卡片可点击 → 跳转 `/points` 和 `/grades` 占位页
- 占位页继承 base.html，显示居中提示 + 返回按钮
- 系统管理卡片所有人可见

## 验证方式

1. 登录 → 欢迎页 3 模块卡片 + 系统管理入口
2. 点击"宿舍管理" → `/dashboard` 仪表盘
3. 点击"积分管理"/"成绩管理" → 占位页"该模块正在开发中"
4. 点击"系统管理" → 学生列表
5. 班主任登录 → 系统管理可见；点击用户管理 → 403
6. 批量删除学生仅 admin 可用
