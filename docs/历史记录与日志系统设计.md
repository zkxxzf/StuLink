# 寝室分配历史记录与系统日志设计

## 1. 概述

系统有两套并行的记录机制：

| 体系 | 表名 | 数据库 | 用途 |
|------|------|--------|------|
| AssignmentHistory | `assignment_history` | `data/history.db`（独立） | 寝室分配变更追溯 |
| OperationLog | `operation_logs` | `data/dormitory.db`（主库） | 全模块运营审计日志 |

**设计原则**：
- 历史库与主库物理隔离，查询历史不影响主业务性能
- 快照模式：记录写入时冗余存储关键字段（姓名、房间名），即使源数据被删除/修改仍可追溯
- 静默失败：记录写入失败绝不阻塞主业务

## 2. AssignmentHistory（分配历史记录）

### 2.1 数据模型

```python
class AssignmentHistory(db.Model):
    __bind_key__ = 'history'       # 写入 data/history.db
    __tablename__ = 'assignment_history'

    id            # 自增主键
    student_id    # 学生 ID（索引）
    student_name  # 学生姓名（快照）
    grade         # 年级（快照）
    class_name    # 班级（快照）
    gender        # 性别（快照）
    room_id       # 房间 ID（索引）
    room_display  # 房间名称（快照，如"A栋 201"）
    bed_number    # 床号（1-8）
    action_type   # 操作类型（索引）
    semester      # 学期（如"2025-2026-2"）
    operator_id   # 操作人 ID
    operator_name # 操作人姓名（快照）
    detail        # 额外上下文（JSON）
    created_at    # 创建时间（索引）
```

### 2.2 action_type 枚举

| 值 | 含义 | 触发操作 |
|----|------|---------|
| `assign` | 分配学生到床位 | 手动拖拽分配 |
| `unassign` | 从床位移除学生 | 取消分配 |
| `move_out` | 学生搬离原床位 | 搬移操作的来源 |
| `move_in` | 学生搬入新床位 | 搬移操作的目标 |
| `swap` | 互换床位 | 互换操作（两人各一条） |
| `auto_assign` | 自动分配 | 一键自动分配 |
| `clear` | 清空班级所有床位 | 清除本班全部分配 |

### 2.3 semester 自动推导

```python
def _current_semester():
    month = datetime.now().month
    year = datetime.now().year
    if 2 <= month <= 7:
        return f"{year-1}-{year}-2"   # 春季学期
    else:
        return f"{year}-{year+1}-1"   # 秋季学期
```

### 2.4 查询示例

```python
# 查询某学生所有历史
AssignmentHistory.query.filter_by(student_id=sid).order_by(desc('created_at')).all()

# 查询某房间当前学期的分配记录
AssignmentHistory.query.filter_by(room_id=rid, semester='2025-2026-2').all()

# 查询某班某操作类型
AssignmentHistory.query.filter_by(grade='2025级', class_name='01班', action_type='assign').all()
```

## 3. 历史记录钩子

### record_assignment() 函数签名

```python
from app.services.history_service import record_assignment

record_assignment(
    action_type='assign',   # 操作类型
    student=student_obj,    # Student 对象
    room=room_obj,          # Room 对象
    bed_number=3,           # 床号
    operator=current_user,  # User 对象
    detail={'from_room': 'B栋 301'}  # 可选，额外上下文
)
```

### 在路由中的插入点

```
assign()    → db.session.commit() 前插入 1 条 record_assignment('assign', ...)
unassign()  → db.session.commit() 前插入 1 条 record_assignment('unassign', ...)
move()      → 搬移前先 snapshot 旧位（move_out），再记录新位（move_in）
swap()      → 生成 2 条 swap 记录，detail 中标注 pair_student
auto_assign() → 遍历分配的 bed，逐条 record_assignment('auto_assign', ...)
clear_class() → 先查询被清除的 bed，逐条 record_assignment('clear', ...)
```

## 4. OperationLog（系统日志）

### 4.1 扩展字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| module | VARCHAR(30) | 'system' | dormitory / system / points / grades / auth |
| severity | VARCHAR(10) | 'INFO' | INFO / WARNING / ERROR |

### 4.2 调用规范

```python
from app.utils.helpers import log_operation

# INFO：常规操作（新增/编辑/删除/导入）
log_operation(current_user, '创建', '学生', sid, f'{name}', module='system')

# WARNING：需要关注的操作（批量删除/清空班级/毕业）
log_operation(current_user, '清空班级', '床位分配', None, detail, 
              module='dormitory', severity='WARNING')

# ERROR：异常操作（一般由异常处理层调用）
# log_operation(None, '异常', '系统', None, traceback, severity='ERROR')
```

### 4.3 模块标识约定

| 模块 | module 值 |
|------|----------|
| 认证 | auth |
| 欢迎页 | welcome |
| 宿舍管理 | dormitory |
| 系统管理 | system |
| 积分管理 | points |
| 成绩管理 | grades |

## 5. 数据库架构

```
data/
├── dormitory.db          ← Flask 主库（业务表 + operation_logs）
└── history.db            ← 历史库（assignment_history 表）
```

config.py 配置：
```python
SQLALCHEMY_BINDS = {
    'history': 'sqlite:///' + os.path.join(BASE_DIR, 'data', 'history.db')
}
```

同一 `db.session.commit()` 会顺序 flush 两个 bind，保证事务一致性。
